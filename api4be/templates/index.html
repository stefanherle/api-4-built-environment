API4BE - Demo Server

Background & Features

The API4BE server implements a hypermedia-driven RESTful API for digital twins of the built environment. It consists of three APIs for the domains of Building Information Modeling (BIM), Geospatial Information Modeling (GIM) and the Internet of Things (IoT). While the GIS and IoT interfaces are covered by established standards, a RESTful interface for IFC models of the BIM domain is defined and implemented. The APIs are interconnected with a federated approach though hypermedia navigation links.

The server offers the following features:

  • Exposing IFC models and their items by a hypermedia-driven RESTful API using objects in JSON format (e.g., ifcJSON, glTF).
  • Interacting with and altering IFC models and their items using HTTP verbs (e.g., POST, DELETE).*
  • Novel methods for the generation and provisioning of GIM layers from georeferenced IFC files through endpoints based on the OGC API - Features.*
  • Modeling sensors in IFC and introducing a linked model approach to their observations (time series and real-time data in the SensorThings API (STA))*

* These features are not fully included or completely ommitted in the open source server.

Examples

The demo server offers a data set and use cases which are described in the paper. It is used to implement and evaluate the capabilities of the server:

The georeferenced duplex model is a planning model of a duplex apartment building in Luxembourg (here). The use case describes the publication of a building during its planning phase. The entire or single parts of the building are exposed as resources through HTTP endpoints for the BIM and the GIM domain.
